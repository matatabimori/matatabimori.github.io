QGIS+VS2019编译中遇到的问题

本篇记录在网上找不到解决方法的一些我遇到的错误。

根据官方指导完成下载步骤
Download and install following packages:

  • CMake
  • GNU flex, GNU bison and GIT with cygwin 32bit or 64bit
  • OSGeo4W 32bit or 64bit
  • ninja

For the QGIS build you need to install following packages from cygwin:

  • bison
  • flex
  • git (even if you already have Git for Windows installed)

and from OSGeo4W (select Advanced Install):

  • qgis-ltr-deps

1. 配置环境变量时使用的VS工具脚本


打开cmake后打开源码目录，configure的时候显示找不到PyQt5，有可能是因为在OSGEO4W中安装的库太多起冲突，重新下载qgis-ltr-deps在别的目录后得到解决。

2. cmake中configure过程中显示找不到EXIV2
添加正确的目录：
EXIV2_INCLUDE_DIR——–>D:/OSGeo4W64/include/exiv2
EXIV2_LIBRARY——–>D:/OSGeo4W64/lib/exiv2.lib
后得到了解决。configure和generate成功。

7.qwt版本不一致导致许多文件出现语法错误

如图以上各种语法错误和各种未定义什么的五六十个报错是因为头文件出了问题，依赖库的目录应该是D:\OSGeo4W64\apps\Qt5\include\qwt6，而程序找到的目录是D:\OSGeo4W64\include\qwt，其中大量的.h文件重名但内容不同，这是大量报错的原因。
将其中的头文件替换一下就得到了解决。

其实也不是因为路径错误了而是应该使用qwt6以上的头文件，而qgis下载的时候下的是qwt5，所以会出现很多未定义的函数和变量之类的报错。
从官网下载了qwt6.1，将src目录里的.h文件替换到了D:\OSGeo4W64\apps\Qt5\include\qwt6，报错消失。

其他问题可参考此篇文章：QGIS+QT5+VS2015（2017）的编译
